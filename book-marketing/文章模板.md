# 文章模板

> 版本：v1.0
> 更新时间：2026-03-01

---

## 模板使用说明

### 如何使用模板

1. 确认文章类型（痛点文/方法文/故事文/观点文）
2. 对应选择模板
3. 按模板结构填充内容
4. 不跳步，不省略

### 四种类型快速判断

| 类型 | 标题特征 | 核心目的 |
|------|----------|----------|
| 痛点文 | "为什么..."、"为什么越..." | 戳中痛点，引发共鸣 |
| 方法文 | "如何..."、"3个方法..." | 提供可执行方案 |
| 故事文 | "那个...的人"、"后来..." | 用故事建立连接 |
| 观点文 | "不是...而是..."、"我发现..." | 表达立场，建立信任 |

## 一、痛点文模板

### 1.1 适用场景

- 读者有痛点，但不知道问题在哪
- 需要帮读者"说出来的话"
- 标题特征："为什么你越努力越焦虑"、"为什么你做得越多领导越看不见"

### 1.2 辩论框架（预设，不用每篇都思考）

```
新手：我好难 / 我很痛苦 / 我不知道怎么办
      ↓
过来人：难在哪？痛苦的是什么？问题出在哪？
      ↓
共识：原来问题在这 / 原来我一直搞错了
```

### 1.3 文章结构

```
【标题】
痛点文标题 = 为什么 + 痛点描述
例：为什么你做得越多，领导越看不见

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【开篇】故事/场景切入（200字）

一个具体的场景，让读者"对号入座"
- 人物：小王/同事/朋友
- 场景：加班/开会/被批评
- 状态：努力但不被认可

示例：
> 小王是我见过最努力的同事。
> 每天第一个到，最后一个走。
> 领导交代的任务，他总是第一时间完成。
> 但年终评优，名单里没有他。
> 他不明白：为什么我做得越多，反而越不被看见？

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【痛点展开】（400字）

1. 这个问题为什么普遍（100字）
   - 列举类似场景
   - 让更多读者有共鸣

2. 读者的真实感受（150字）
   - 描述心理状态
   - "你可能会觉得..."
   - 不要批判，要理解

3. 为什么会这样（150字）
   - 表面原因
   - 先不说深层原因，留到下一部分

示例：
> 小王的困惑，很多人都有。
> 你可能也有过类似的经历：
> 加班到深夜，领导只会说"辛苦了"；
> 做了大量工作，汇报时却说不出亮点；
> 看着那些看似不如自己的人升职，心里不是滋味。
> 
> 你会觉得委屈：我明明这么努力。
> 你会觉得不公平：为什么结果是这样？
> 你甚至会怀疑：是不是我不够好？
> 
> 但问题可能不是你不够努力。
> 而是你努力的方向，错了。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【深层分析】（400字）

1. 表面原因 vs 深层原因（200字）
   - 表面：努力不够、运气不好
   - 深层：方向错了、方法错了

2. 一个反直觉的观点（200字）
   - 打破读者的认知
   - "真正的问题是..."

示例：
> 很多人以为，努力就够了。
> 只要我做得够多，总有一天会被看见。
> 
> 但职场的真相是：努力只是底线，不是加分项。
> 
> 你做得再多，如果不能被"看见"和"记住"，就是无效努力。
> 
> 就像小王，他每天加班，但领导只看到结果，看不到过程。
> 他做了很多事，但每件事都没有突出亮点。
> 他埋头苦干，但从不主动汇报和展示。
> 
> 真正的问题，不是他不够努力。
> 而是他没有学会"让努力被看见"。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【行动建议】（300字）

1-2个可执行的小步骤，不贪多

结构：
- 第一步：做什么（具体动作）
- 第二步：怎么做（简单方法）
- 注意事项：避坑

示例：
> 如果你也和小王一样，试试这两件事：
> 
> 第一，每周给领导发一份简短的周报。
> 不用很长，列出3件事：这周完成了什么、下周计划做什么、需要什么支持。
> 让领导知道你在做什么，比闷头做更重要。
> 
> 第二，每完成一个重要任务，主动汇报结果。
> 不要等领导来问，而是主动说：这件事我做完了，结果是这样的。
> 
> 从今天开始，别只顾着努力。
> 也想想，怎么让努力被看见。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【结尾金句】（100字）

一句可以转发的话

示例：
> 努力，只是职场的入场券。
> 让努力被看见，才是升职加薪的通行证。
```

### 1.4 痛点文写作要点

| 要点 | 说明 |
|------|------|
| 开篇要具体 | 不要泛泛而谈，要有一个具体的人/场景 |
| 共情在前 | 先理解读者，不要急着批判 |
| 分析在后 | 给出深层原因，打破认知 |
| 行动要少 | 1-2个就够了，不要贪多 |

## 二、方法文模板

### 2.1 适用场景

- 读者有明确的问题，需要解决方案
- 标题特征："如何..."、"3个方法..."、"怎么做..."

### 2.2 辩论框架（预设）

```
新手：怎么办 / 我要怎么做
      ↓
过来人：试试这三步 / 方法是这样的
      ↓
共识：原来可以这样做 / 我试试看
```

### 2.3 文章结构

```
【标题】
方法文标题 = 数字 + 方法 + 收益
例：3个方法，让你不再无效加班

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【开篇】痛点快速切入（150字）

用1-2句话点出问题，快速进入正题
不要展开，直奔主题

示例：
> 你是不是经常加班到很晚，却发现重要的工作还没做完？
> 每天忙得团团转，但效率却不高？
> 
> 这篇文章，给你3个方法，让你摆脱无效加班。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【方法核心】（800字）

分3-5个步骤，每个步骤包含：

### 方法一：XXX

1. 是什么（概念）
   用一句话解释这个方法

2. 为什么有效（原理）
   用1-2句话说明背后的逻辑

3. 怎么做（具体操作）
   给出具体的步骤，读者照做就行

4. 注意事项（避坑）
   可能的误区或陷阱

示例：
### 方法一：列三件事

是什么？
> 每天早上，列出今天必须完成的三件事。
> 注意，是三件，不是十件。

为什么有效？
> 人的精力是有限的。
> 列太多，会分散注意力，最后每件事都做不好。
> 只列三件，可以让你专注最重要的事。

怎么做？
> 第一步：打开备忘录或纸笔。
> 第二步：问自己，今天最重要的三件事是什么？
> 第三步：把这三件事写下来，放在显眼的位置。
> 第四步：先做第一件，做完再做第二件。

注意事项？
> 不要把"回邮件"、"开会"这种琐事列进去。
> 三件事，应该是推进重要工作的，不是完成日常杂务。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【案例佐证】（300字）

1-2个真实案例，证明方法有效

示例：
> 我用这个方法已经两年了。
> 
> 以前我每天列十件事，结果一天下来常常只完成一半。
> 后来改成只列三件，反而效率更高了。
> 
> 我有个同事也是，以前天天加班到晚上十点。
> 用了这个方法后，现在基本能在七点前下班。
> 他说，最大的改变不是工作时间变少了，而是知道自己每天要做什么。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【行动清单】（200字）

3条以内，立刻能做的

示例：
> 今天就开始试试：
> 
> 1. 明天早上，列出你最重要的三件事
> 2. 先做完第一件，再想第二件
> 3. 一周后，看看效率有没有提升

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【结尾鼓励】（100字）

给读者信心

示例：
> 方法很简单，难的是坚持。
> 从今天开始，给自己一个机会。
> 也许一周后，你会发现，加班不再是常态。
```

### 2.4 方法文写作要点

| 要点 | 说明 |
|------|------|
| 开篇要快 | 不要铺垫，直奔主题 |
| 方法要具体 | 读者照做就能执行 |
| 步骤要清晰 | 第一步、第二步、第三步 |
| 案例要真实 | 用自己或身边人的例子 |

## 三、故事文模板

### 3.1 适用场景

- 需要用故事引发共鸣
- 标题特征："那个...的人"、"后来怎样了"

### 3.2 辩论框架（预设）

```
旁观者：后来呢 / 结果怎么样
      ↓
主角：我学会了 / 我明白了
      ↓
共识：值得 / 有启发
```

### 3.3 文章结构

```
【标题】
故事文标题 = 主角 + 核心冲突
例：那个35岁被裁的同事，后来怎样了

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【故事开场】（300字）

人物、背景、初始状态

包含：
- 人物：谁
- 背景：什么环境
- 初始状态：最初是什么样的

示例：
> 35岁那年，老张被裁了。
> 
> 他在这家公司干了十年，从普通员工做到部门主管。
> 薪资不错，团队稳定，日子过得顺风顺水。
> 
> 他以为，可以一直这样干到退休。
> 
> 直到那天，HR找他谈话。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【冲突展开】（400字）

遇到了什么问题、挣扎、尝试

包含：
- 事件：发生了什么
- 挣扎：内心的冲突
- 尝试：做了什么努力

示例：
> 那天之后，老张开始疯狂投简历。
> 
> 但他很快发现，35岁的职场，没有他想的那么友好。
> 
> 投了50份简历，面试邀约只有3个。
> 面试官看着他的履历，问得最多的是：
> "你这个年纪，还能加班吗？"
> "你为什么不考虑管理岗位？"
> "你的薪资期望能不能降一降？"
> 
> 老张第一次感到，年龄成了他的"原罪"。
> 
> 他开始失眠，开始怀疑自己。
> 是不是我真的不行了？
> 是不是职场对35岁的人，就是不公平？

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【转折点】（300字）

一个关键的觉悟或契机

包含：
- 触发点：什么让他改变
- 觉悟：他想明白了什么
- 行动：他开始做什么

示例：
> 直到有一天，他遇到了以前的同事。
> 
> 那个同事比他还大两岁，但一年前就主动离职了。
> 现在做自由职业，收入比上班还高。
> 
> 同事问他：你有没有想过，离开这家公司，你能做什么？
> 
> 这句话点醒了老张。
> 
> 他突然意识到，自己从来没有认真想过这个问题。
> 十年来，他只是在"上班"，不是在"成长"。
> 
> 他的技能，只适用于这家公司。
> 他的资源，都在这个圈子里。
> 
> 一旦离开，他什么都没有。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【结局与感悟】（300字）

结果如何、学到了什么

包含：
- 结果：后来怎么样了
- 感悟：学到了什么
- 读者启发：给读者什么启示

示例：
> 后来，老张没有继续找工作。
> 
> 他用自己十年的行业经验，开始做咨询。
> 起初很难，但慢慢地，客户越来越多。
> 
> 现在的他，收入比上班时高了30%，而且时间自由。
> 
> 他告诉我：被裁，反而是他人生的转折点。
> 如果不是那次变故，他可能永远不会跳出舒适圈。
> 
> 他说，35岁最大的教训是：
> 不要等到被迫改变，才开始改变。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【金句收尾】（100字）

从故事提炼的道理

示例：
> 职场没有永远的稳定。
> 
> 真正的安全感，不是来自公司，而是来自你自己。
> 你能创造的价值，才是你的护城河。
```

### 3.4 故事文写作要点

| 要点 | 说明 |
|------|------|
| 人物要具体 | 有名字、有背景 |
| 冲突要真实 | 让读者有代入感 |
| 转折要自然 | 不是突然"顿悟"，而是有触发点 |
| 感悟要简洁 | 不要说教，让故事自己说话 |

## 四、观点文模板

### 4.1 适用场景

- 需要表达立场、建立信任
- 标题特征："不是...而是..."、"我发现..."、"...的真相"

### 4.2 辩论框架（预设）

```
质疑者：凭什么这么说 / 我不信
      ↓
拥护者：因为... / 事实是这样的
      ↓
共识：有道理 / 我认同
```

### 4.3 文章结构

```
【标题】
观点文标题 = 鲜明的观点
例：35岁后，拼的不是努力

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【观点抛出】（200字）

开门见山，抛出核心观点
不要铺垫，直接亮观点

示例：
> 我发现一个很残酷的真相。
> 
> 35岁以后，努力变得越来越不值钱。
> 
> 你可能会觉得扎心，但这是事实。
> 
> 年轻的时候，努力是优势。因为大部分人都不够努力。
> 但到了35岁，努力只是底线。周围的人都在努力，你已经不稀奇了。
> 
> 真正拉开差距的，是另外三件事。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【论据支撑】（600字）

分3个论据，每个论据：

1. 论据一：数据/案例
   用事实支撑观点

2. 论据二：逻辑推演
   从逻辑上论证

3. 论据三：反面论证
   从反面证明观点

示例：
### 论据一：能力大于努力

> 我见过很多35岁的人，依然在拼体力。
> 加班、熬夜、比别人多做。
> 
> 但那些升职快的人，不是最努力的。
> 而是最会思考的。
> 
> 他们知道什么工作有价值，什么是无效劳动。
> 他们知道怎么用更少的时间，做出更好的结果。
> 
> 努力，只能让你不落后。
> 能力，才能让你领先。

### 论据二：选择大于努力

> 有个朋友，30岁时跳槽到了一家小公司。
> 当时很多人不理解，觉得他"不思进取"。
> 
> 结果那家公司三年后上市了，他成了总监。
> 
> 如果他留在原来的大公司，现在可能还是个普通经理。
> 
> 同样的努力，不同的选择，结果天差地别。
> 
> 35岁以后，方向比速度更重要。

### 论据三：资源大于努力

> 以前我觉得，靠人脉是"走后门"。
> 
> 后来才发现，真正厉害的人，都在经营人脉。
> 
> 你认识谁，往往比你会什么更重要。
> 
> 不是让你去拍马屁，而是：
> 建立真正的连接，创造互惠的价值。
> 
> 35岁以后，你的资源池，决定了你的天花板。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【回应质疑】（300字）

预判反对意见，提前回应

示例：
> 你可能会说：那努力就没用了吗？
> 
> 当然有用。努力是基础。
> 
> 我想说的是，35岁以后，光有努力是不够的。
> 
> 你需要在努力的同时，学会思考、学会选择、学会经营。
> 
> 努力，是入场券。
> 但想走得远，光有入场券是不够的。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【行动号召】（200字）

读者看完应该怎么做

示例：
> 如果你已经35岁，或者快到了：
> 
> 1. 停下来想一想，你的努力方向对吗？
> 2. 问自己一个问题：如果离开现在的公司，你还有什么？
> 3. 开始建立你的能力、选择、资源池。
> 
> 不要等到被动改变，才开始行动。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【结尾升华】（100字）

观点的延伸意义

示例：
> 35岁，是人生的分水岭。
> 
> 以前，拼的是谁更努力。
> 以后，拼的是谁更清醒。
> 
> 愿你，做那个清醒的人。
```

### 4.4 观点文写作要点

| 要点 | 说明 |
|------|------|
| 观点要鲜明 | 不要模棱两可 |
| 论据要充分 | 数据、案例、逻辑都要有 |
| 质疑要回应 | 预判反对意见，提前反驳 |
| 行动要具体 | 给读者一个明确的入口 |
