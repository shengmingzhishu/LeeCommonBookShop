## 三、写作风格指南

### 3.1 人设
- **角色**：过来人 / 朋友
- **语气**：真诚、不装、有温度
- **距离感**：像跟老朋友聊天，不像说教

### 3.2 写作风格
- **字数**：1500-2000字
- **结构**：故事开场 → 痛点共鸣 → 方法/观点 → 行动建议
- **节奏**：短句为主，每段不超过4行，多用空行

### 3.3 禁忌
- ❌ 不说教、不爹味
- ❌ 不用网络烂梗
- ❌ 不堆砌金句
- ❌ 不贩卖焦虑
- ❌ 不标题党

### 3.4 必须有
- ✅ 至少1个真实故事或案例
- ✅ 至少1句可转发的金句
- ✅ 结尾有行动建议或思考题

## 四、文章模板

### 4.1 痛点文模板

```
【标题】戳中痛点，引发共鸣

【开篇】故事/场景切入（200字）
一个具体的场景或故事，让读者"对号入座"

【痛点展开】（400字）
- 这个问题为什么普遍
- 读者的真实感受
- 为什么会这样

【深层分析】（400字）
- 表面原因 vs 深层原因
- 一个反直觉的观点

【行动建议】（300字）
- 1-2个可执行的小步骤
- 不贪多，给一个明确的入口

【结尾金句】（100字）
一句可以转发的话
```

**辩论框架**：
- 新手："我好难" → 过来人："难在哪" → 共识："原来问题在这"

### 4.2 方法文模板

```
【标题】承诺价值，不夸张

【开篇】痛点快速切入（150字）
用1-2句话点出问题，快速进入正题

【方法核心】（800字）
分3-5个步骤，每个步骤：
- 是什么（概念）
- 为什么有效（原理）
- 怎么做（具体操作）
- 注意事项（避坑）

【案例佐证】（300字）
1-2个真实案例，证明方法有效

【行动清单】（200字）
3条以内，立刻能做的

【结尾鼓励】（100字）
给读者信心
```

**辩论框架**：
- 新手："怎么办" → 过来人："三步走" → 共识："试试看"

### 4.3 故事文模板

```
【标题】故事主角+核心冲突

【故事开场】（300字）
人物、背景、初始状态

【冲突展开】（400字）
遇到了什么问题、挣扎、尝试

【转折点】（300字）
一个关键的觉悟或契机

【结局与感悟】（300字）
结果如何、学到了什么

【金句收尾】（100字）
从故事提炼的道理
```

**辩论框架**：
- 旁观者："后来呢" → 主角："我学会了" → 共识："值得"

### 4.4 观点文模板

```
【标题】鲜明的观点/立场

【观点抛出】（200字）
开门见山，抛出核心观点

【论据支撑】（600字）
- 论据1：数据/案例
- 论据2：逻辑推演
- 论据3：反面论证

【回应质疑】（300字）
预判反对意见，提前回应

【行动号召】（200字）
读者看完应该怎么做

【结尾升华】（100字）
观点的延伸意义
```

**辩论框架**：
- 质疑者："凭什么" → 拥护者："因为" → 共识："认同"

## 五、素材库框架

### 5.1 素材类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 案例 | 真实或改编的故事 | 那个35岁被裁的同事... |
| 数据 | 权威报告、调研 | 某报告显示35岁后跳槽成功率下降40% |
| 金句 | 可转发的句子 | 35岁前靠努力，35岁后靠选择 |
| 观点 | 有洞察的论断 | 真正的稳定，不是不跳槽 |
| 反例 | 对比论证 | 那些频繁跳槽的人... |
| 名人 | 名人名言、故事 | 巴菲特说... |

### 5.2 素材库结构

每个板块素材库包含：
```
## 案例（5-10个）
## 数据（5-10条）
## 金句（10-20句）
## 观点（5-10条）
## 反例（3-5个）
## 名人素材（3-5个）
```

### 5.3 素材来源

- 公众号爆款文章（拆解学习）
- 书籍摘录
- 新闻热点
- 知乎高赞
- 个人经历/观察

## 七、多角色讨论机制

### 7.1 适用场景

| 文章类型 | 是否使用多角色讨论 | 说明 |
|----------|-------------------|------|
| 痛点文 | 否 | 直接套模板 |
| 方法文 | 否 | 直接套模板 |
| 故事文 | 否 | 直接套模板 |
| 观点文 | ✅ 是 | 需要深度思考，多角度论证 |

**重点文章定义**：观点文，共27篇，使用多角色讨论

### 7.2 角色定义

| 角色 | 职责 | 特点 |
|------|------|------|
| 提问者 | 代表读者提问、质疑 | 通俗易懂，直击要害 |
| 过来人 | 分享经验、给建议 | 有深度，有洞察 |
| 观察者 | 总结提炼、升华观点 | 抽离全局，提炼本质 |

### 7.3 讨论回合设计

**固定2回合，每回合3句话**

```
【第1回合】确定核心观点

提问者：这篇文章要解决读者的什么困惑？
过来人：我觉得核心问题是...，很多读者...
观察者：总结一下，核心观点是...

【第2回合】预判质疑与回应

提问者：读者看到这个观点，可能会质疑什么？
过来人：他们可能会说...，但实际情况是...
观察者：所以我们在文章中要回应这一点...

↓ 讨论结束，开始写作
```

### 7.4 Token优化策略

**核心原则：讨论过程内部模拟，不输出**

```
传统方式（token高）：
┌──────────────────────────────────────┐
│ 提问者：...（输出）                   │
│ 过来人：...（输出）                   │
│ 观察者：...（输出）                   │
│ 提问者：...（输出）                   │
│ 过来人：...（输出）                   │
│ 观察者：...（输出）                   │
│ ↓ 6段输出                             │
└──────────────────────────────────────┘

优化方式（token低）：
┌──────────────────────────────────────┐
│ 【内部模拟2回合讨论】                  │
│ （不输出，只在思考中进行）             │
│ ↓                                     │
│ 直接输出：讨论结果 + 文章正文          │
│ ↓ 1段输出                              │
└──────────────────────────────────────┘
```

### 7.5 讨论协议

**讨论触发条件**：
- 文章类型为"观点文"
- 标题清单中标注的观点文

**讨论执行方式**：
- 指挥官在写作前，内部模拟2回合讨论
- 不向用户输出讨论过程
- 讨论结果融入文章中

**讨论结果体现**：
- 核心观点更清晰
- 预判质疑并提前回应
- 论证更有层次

### 7.6 效果对比

| 方式 | Token消耗 | 文章质量 | 适用 |
|------|-----------|----------|------|
| 无讨论 | 最低 | 标准 | 普通73篇 |
| 2回合讨论（内部模拟） | 低 | 高 | 重点27篇 |
| 多回合讨论（输出） | 高 | 更高 | 不推荐 |

## 八、质量保证

### 8.1 质量标准

| 维度 | 标准 |
|------|------|
| 内容 | 有价值、可落地、不空洞 |
| 结构 | 逻辑清晰、过渡自然 |
| 语言 | 通顺、简洁、有温度 |
| 原创性 | 不抄袭、可借鉴观点但要有自己的表达 |
| 可读性 | 短句、短段、有空行 |

### 8.2 质量检查流程

```
初稿完成
    ↓
自检（清单check）
    ↓
内部润色
    ↓
交付
    ↓
用户review
    ↓
修正
```

### 8.3 返工规则

- 用户指出问题 → 指挥官判断问题类型 → 对应角色修正
- 严重问题（方向错误）：重写
- 一般问题（表达不佳）：润色
- 格式问题：直接修正

## 九、Token优化策略

### 9.1 核心原则

**一次投入，批量复用**

### 9.2 具体策略

| 策略 | 说明 | 效果 |
|------|------|------|
| 模板预设 | 4种类型模板提前写好 | 不用每篇都思考结构 |
| 辩论框架预设 | 每种类型的辩论结果写进模板 | 不用每篇都辩论 |
| 素材库预设 | 每个板块预设10-20个素材 | 不用每篇都想素材 |
| 批量自检 | 用清单check而非逐篇检查 | 检查消耗降到最低 |
| 分批交付 | 每批10篇，减少交互次数 | 总交互约20次 |

### 9.3 Token消耗估算

| 方案 | 总交互 | 预估消耗 |
|------|--------|----------|
| 原方案（每篇7步） | 700次 | 极高 |
| 优化方案v2 | 50次 | 中等 |
| **优化方案v3（本方案）** | **约20次** | **最低** |
