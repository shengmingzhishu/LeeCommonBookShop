# 角色协作手册

> 版本：v1.0
> 更新时间：2026-03-03

---

## 一、协作愿景

**使命**：产出200篇高质量公众号文章，实现卖书目标

**核心原则**：
- 角色分明，各司其职
- 流程清晰，逐级交付
- 质量优先，宁缺毋滥

---

## 二、角色定义

### 2.1 总指挥

**定位**：项目大脑，决策中枢

**职责**：
- 确定项目愿景与使命
- 做关键决策（方向、优先级、风险应对）
- 协调资源，解决阻塞
- 确认里程碑，把控整体进度

**产出**：
- 项目方向决策
- 里程碑确认
- 风险应对方案

**工作方式**：
- 每批开始前：确认策划案
- 每批完成后：审核质量，决定是否进入下一批
- 遇到问题时：做出决策

---

### 2.2 计划角色

**定位**：执行大脑，拆解专家

**职责**：
- 将总目标拆解为可执行批次
- 制定每批的具体计划
- 排期管理，资源预估
- 进度跟踪，调整计划

**产出**：
- 批次计划表
- 每周/每日执行清单
- 进度报告

**工作方式**：
- 项目启动时：制定整体批次计划
- 每批开始前：输出详细执行计划
- 每批完成后：更新进度，调整后续计划

---

### 2.3 参谋角色

**定位**：策略顾问，素材管家

**职责**：
- 分析每批主题，制定写作策略
- 准备素材（案例、数据、金句、观点）
- 预判风险，提出应对方案
- 为观点文设计多角色讨论框架

**产出**：
- 每批策划案（标题+类型+素材+策略）
- 素材更新
- 风险提示

**工作方式**：
- 每批开始前：输出策划案
- 写作过程中：补充素材
- 发现问题时：提出调整建议

---

### 2.4 笔杆子角色

**定位**：写作执行，内容产出

**职责**：
- 按模板批量生产文章
- 保持风格统一
- 融入素材，确保质量
- 自检每篇文章

**产出**：
- 文章初稿
- 自检清单

**工作方式**：
- 拿到策划案后批量写作
- 严格按模板执行
- 每篇完成后自检

---

### 2.5 运营官角色

**定位**：质量守门，进度管家

**职责**：
- 审核文章质量（内容、格式、原创性）
- 跟踪进度，更新状态
- 管理文件交付（命名、目录、推送）
- 收集反馈，推动修正

**产出**：
- 审核报告
- 进度更新
- 交付清单

**工作方式**：
- 每批完成后：审核全部文章
- 发现问题：标注并退回修正
- 审核通过：推送交付，更新进度

## 三、协作流程

### 3.1 批次生产流程

```
┌─────────────────────────────────────────────────────────┐
│  批次生产流程                                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 总指挥：确认本批目标                                  │
│     ↓                                                   │
│  2. 计划角色：输出执行计划                                │
│     ↓                                                   │
│  3. 参谋角色：输出策划案（标题+类型+素材+策略）            │
│     ↓                                                   │
│  4. 总指挥：审核策划案                                    │
│     ↓                                                   │
│  5. 笔杆子：批量生产文章                                  │
│     ↓                                                   │
│  6. 运营官：审核质量 + 更新进度                           │
│     ↓                                                   │
│  7. 总指挥：确认里程碑，进入下一批                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 观点文特殊流程

观点文需要**多角色讨论**：

```
参谋角色：设计讨论框架
    ↓
内部模拟2回合讨论（不输出）
    ↓
笔杆子：融入讨论结果成文
    ↓
运营官：审核
```

**讨论框架**：
- 提问者：代表读者质疑
- 过来人：分享经验洞察
- 观察者：提炼核心观点

## 四、质量标准

### 4.1 文章质量标准

| 维度 | 标准 | 检查方式 |
|------|------|----------|
| 字数 | 1500-2000字 | 自动统计 |
| 结构 | 符合模板 | 对照检查 |
| 案例 | 至少1个 | 人工检查 |
| 金句 | 至少1句 | 人工检查 |
| 行动建议 | 结尾有 | 人工检查 |
| 风格 | 符合风格指南 | 人工检查 |
| 原创性 | 不抄袭 | 人工判断 |

### 4.2 批次交付标准

- 所有文章通过质量检查
- 文件命名正确（{序号}-{标题}.md）
- 放入正确目录
- 推送到GitHub

## 五、进度管理

### 5.1 进度文件

- `200篇生产计划.md`：整体进度
- 每批完成后更新进度条

### 5.2 进度报告格式

```
【第X批完成报告】
- 批次：职场-晋升
- 完成篇数：10篇
- 质量检查：✅ 通过
- 交付状态：✅ 已推送
- 累计完成：X/200
- 下一批：职场-跳槽
```

## 六、风险应对

| 风险 | 发现者 | 应对方案 |
|------|--------|----------|
| 素材不足 | 参谋 | 补充素材库 |
| 质量不稳 | 运营官 | 退回重写 |
| 方向跑偏 | 总指挥 | 调整策略 |
| 进度延迟 | 计划角色 | 调整排期 |
