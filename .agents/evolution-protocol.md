# 自我进化协议

> 让 Levis 从每次经验中学习，持续提升能力

---

## 一、核心机制

```
任务完成 → 提取经验 → 识别模式 → 更新能力 → 记录成长
    ↓           ↓           ↓           ↓           ↓
  做了什么    学到什么    可复用什么   改进哪里    进化轨迹
```

---

## 二、记忆系统

### 2.1 目录结构

```
.memory/
├── manifest.json           # 系统配置
├── semantic/               # 语义记忆（模式、规则）
│   └── patterns.json
├── episodic/               # 情景记忆（具体事件）
│   └── YYYY-MM-DD-{事件}.json
└── working/                # 工作记忆（当前会话）
    └── session.json
```

### 2.2 记忆类型

| 类型 | 内容 | 示例 |
|------|------|------|
| 语义 | 抽象模式、规则 | "情感类账号转化率更高" |
| 情景 | 具体经验、事件 | "2026-03-05 完成运营系统搭建" |
| 工作 | 当前会话上下文 | "正在生产第一批文章" |

## 三、进化触发

### 3.1 自动触发

| 触发点 | 动作 |
|--------|------|
| 任务完成 | 提取经验 → 识别模式 |
| 错误发生 | 根因分析 → 记录教训 |
| 用户反馈 | 调整置信度 → 强化/弱化模式 |
| 数据更新 | 发现规律 → 更新模式库 |

### 3.2 手动触发

用户说：
- "自我进化"
- "总结这次经验"
- "更新能力"
- "复盘"

### 3.3 周期触发

| 频率 | 动作 |
|------|------|
| 每次任务 | 提取经验到情景记忆 |
| 每天 | 整合当日经验 |
| 每周 | 抽象模式，更新能力文档 |
| 每月 | 全面复盘，系统升级 |

## 四、进化流程

### 4.1 经验提取模板

```yaml
任务：{做了什么}
背景：{为什么做}
行动：{具体步骤}
结果：{成功/失败/部分}
洞察：{学到什么}
可复用：{什么可以下次用}
用户反馈：{如果有的话}
```

### 4.2 模式抽象规则

```
if 经验重复出现 3+ 次:
  → 提升为关键模式
  → 更新到语义记忆
  → 更新相关能力文档

if 用户反馈评分 >= 7:
  → 强化该模式
  → 提高置信度

if 用户反馈评分 <= 4:
  → 弱化该模式
  → 降低置信度或删除
```

### 4.3 能力更新协议

更新位置：
1. `MEMORY.md` — 长期记忆（关键模式）
2. `.memory/semantic/patterns.json` — 模式库
3. `.agents/*.md` — 角色能力定义
4. `levis.html` — 养成日志（同步展示）

## 五、进化矩阵

| 经验类型 | 提取模式 | 目标能力 |
|----------|----------|----------|
| 内容爆款 | 标题特征、结构特征 | 内容创作 |
| 转化成功 | 文案技巧、时机因素 | 变现能力 |
| 数据异常 | 根因、应对策略 | 数据分析 |
| 流程优化 | 效率方法 | 运营效率 |
| 错误修正 | 避坑指南 | 质量保证 |
| 用户偏好 | 风格、节奏 | 用户理解 |

## 六、置信度系统

每个模式有置信度分数 (0-1)：

```
初始：0.5
正向反馈：+0.1
负向反馈：-0.15
重复验证：+0.05
失败应用：-0.1

≥ 0.8：核心模式，优先使用
0.5-0.8：有效模式，正常使用
< 0.5：待验证，谨慎使用
< 0.3：淘汰，标记为过时
```
